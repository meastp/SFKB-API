Oppdateringsgrensesnitt for SFKB

Grov oversikt over funksjonalitet:

  • Hente liste over tilgjengelige datasett
  • Hente metadata for et bestemt datasett
  • Hente data fra et bestemt datasett
    • Med lesetilgang eller skrivetilgang (medfører låsing)
      • områdebegrensning
      • egenskapsspørring (begrenset i første versjon til bygningsnummer eller lokalid)
  • Lagre data til et bestemt datasett
    • Operasjoner som håndteres: nytt objekt, endre objekt og slett objekt

get /datasets/{datasetId}/features
Hent ut features fra et dataset (getData)
Henter en liste over alle dataset som brukeren har lese- eller skrivetilgang til.

Path parameters

datasetId (required)
Path Parameter — Numeric ID of the dataset to get format: int32

Query parameters

lock (optional)
Query Parameter — TODO: Beskrive låsing
bbox (optional)
Query Parameter — <p>Henter ut objekter i et bestemt område (angitt med et rektangel)</p> <p>Dette kan være aktuelt for å få ut alle objekter som ikke er direkte knyttet til et objekt, som f.eks mønelinje og takkant til en bygning.</p> <p>Eksempel: Hent ut et 10x10m rektangel</p> <p><strong>TODO: Usikker på syntaks</strong></p> <p>?bbox=(10,10,20,20)</p>
query (optional)
Query Parameter — <p>Filter på egenskaper og/eller objekttype for å begrense hvilke features som hentes ut.</p> <p>Dersom objekttypen ikke skal begrenses, kan den angis med <code>*</code></p> <p>Eksempel: Hent en bygning med bestemt bygningsnummer</p> <p><code>?query=eq(Bygning/bygningsnummer,1234)</code></p> <p>Eksempel: Hent et objekt med bestemt lokalid</p> <p><code>?query=eq(*/identifikasjon/lokalid,000a7b9b-7088-482f-a3cf-9761b97a54b6)</code></p> <p>Merk: Når et objekt hentes ut med lokalid er det normalt ikke nødvendig å filtrere på objekttype, derfor er objekttypen angitt med <code>*</code>.</p> <p>Merk: Spørringer er foreløpig begrenset til eksemplene over, nemlig spørringer etter bygningsnummer og lokalid.</p> <p>Generell regel for egenskapsspørringer:</p> <p>For å angi hvilken objekttype og hvilket attributt (full struktur med struct-/liste-egenskaper) det spørres etter, bruker vi</p> <blockquote> <p>&lt;objekttypenavn&gt;/&lt;attributtnavn&gt; [ / &lt;attributtnavn&gt; …]</p> </blockquote> <p>Hvis vi ikke ønsker å begrense spørringen til en bestemt objekttype, kan vi angi * som <objekttypenavn>.</p>
